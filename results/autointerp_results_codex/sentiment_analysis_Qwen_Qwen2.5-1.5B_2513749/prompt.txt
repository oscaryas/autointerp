You are tasked with conducting systematic probing experiments on the language model `Qwen/Qwen2.5-1.5B`.

## Objective
Your goal is to understand what features the model has learned by training linear probes on its internal representations. Specifically, you will probe for: sentiment (positive vs negative).

## Task Description
Sentiment Analysis Probing

Task: Probe whether the model internally represents sentiment by measuring whether it can correctly classify sentiment from text.

Behavioral Labeling (you must do this first):
For each example in labeled_data.jsonl, run the model to classify the sentiment of the text (positive or negative). Compare the model's prediction to the `ground_truth_sentiment` field (0=negative, 1=positive). Assign:
- label=1 if the model correctly identifies the sentiment
- label=0 if the model misclassifies the sentiment
Save these behavioral labels to `behavioral_labels.jsonl`.

Objective: Train linear probes on model activations to predict whether the model will correctly classify the sentiment of a given text. This reveals which layers encode sentiment information most strongly.

Questions to answer:
1. Which layers best predict whether the model will classify sentiment correctly?
2. Which token positions are most informative (first, last, mean-pooled, max-pooled)?
3. How accurately can a simple linear probe predict the model's sentiment classification behavior?

Dataset: 1000 IMDB movie reviews (500 positive, 500 negative). The `ground_truth_sentiment` field gives the true label for comparison.

Expected Analysis:
- Probe all layers of the model
- Test different token aggregation strategies
- Use cross-validation for robust estimates
- Report accuracy, precision, recall, F1, ROC-AUC
- Visualize probe performance across layers

Deliverables:
1. behavioral_labels.jsonl — your assigned labels based on model behavior
2. Trained probe saved to `final_probe/` directory
3. Metrics in JSON format
4. Visualizations of results (PNG/PDF)
5. Written analysis of findings

## Data
You have access to `labeled_data.jsonl` containing 1000 examples. Depending on the task, this file either contains pre-assigned labels (`"label"` field) or requires you to determine behavioral labels yourself by running the model. See the Task Description above for which applies.

If you determine behavioral labels, save them to `behavioral_labels.jsonl` in this format:
```
{{"text": "example text", "label": 0}}
```
Use `label=1` for the positive class and `label=0` for the negative class as defined in the Task Description.
